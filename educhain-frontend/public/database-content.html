What is a Database?

A database is an organized collection of data that can be easily accessed, managed, and updated. Databases are used to store information in a structured way.

Key Points:
  • Databases store data persistently
  • Data is organized in tables with rows and columns
  • Databases support concurrent access by multiple users
  • Data integrity and security are built-in features

Types of Databases

  • Relational Databases: Use tables with relationships (MySQL, PostgreSQL)
  • NoSQL Databases: Flexible data models (MongoDB, Cassandra)
  • Graph Databases: Store data as nodes and relationships (Neo4j)

Basic SQL Commands

Creating a Table

CREATE TABLE students ( id INTEGER PRIMARY KEY, name VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE, age INTEGER, enrollment_date DATE );

Inserting Data

INSERT INTO students (name, email, age, enrollment_date) VALUES ('John Doe', 'john@example.com', 20, '2024-01-15');

Querying Data

SELECT name, email, age FROM students WHERE age >= 18 ORDER BY name;

ðŸ”— Lesson 3: Database Relationships

Primary Keys and Foreign Keys

  • Primary Key: Unique identifier for each record
  • Foreign Key: References primary key in another table
  • Relationships: One-to-One, One-to-Many, Many-to-Many

Example: Students and Courses

-- Students table CREATE TABLE students ( student_id INTEGER PRIMARY KEY, name VARCHAR(100) ); -- Courses table CREATE TABLE courses ( course_id INTEGER PRIMARY KEY, title VARCHAR(200), instructor_id INTEGER ); -- Enrollments table (junction table) CREATE TABLE enrollments ( student_id INTEGER, course_id INTEGER, grade VARCHAR(2), FOREIGN KEY (student_id) REFERENCES students(student_id), FOREIGN KEY (course_id) REFERENCES courses(course_id) );
